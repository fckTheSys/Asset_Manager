# TankToolBox Roadmap

## Stability

- Keep `Self check / diagnostics` as the first support step for Redshift/CentiLeo node-space problems.
- Replace placeholder `PLUGIN_ID` before distributing the plugin outside local testing.
- Keep report files (`ttb_report.txt`, `ttb_asset_browser_report.txt`) as lightweight production logs.

## Asset Browser

- Improve validation rules for project-specific naming conventions.
- Add optional copy/collect texture operation if the pipeline needs portable asset packages.
- Add persistent UI settings after the main workflows stabilize.

## Materials

- Expand CentiLeo node IDs when new builds expose stable asset IDs.
- Add update policies beyond `Update existing only` if needed: create missing, update only selected map types, or preserve manual edits.
